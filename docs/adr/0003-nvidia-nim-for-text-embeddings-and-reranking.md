# NVIDIA NIM replaces self-hosted models for text embeddings and reranking

> This decision covers the full migration (issue #76), landing incrementally across #77 (embedding provider), #78 (reranker provider), #79 (reindex + validation), and #80 (removal of the self-hosted providers). Some consequences below only take effect once the later tickets land — each is noted.

The hot path (`embed_query` + rerank) ran self-hosted BGE-M3 and BGE-Reranker/Qwen-Reranker on local `mps`, which was the bottleneck for query latency. We migrate text embeddings and reranking to NVIDIA NIM (`nvidia/llama-3.2-nv-embedqa-1b-v2` and `nvidia/llama-3.2-nv-rerankqa-1b-v2` — NIM's low-latency 1B generation), reusing the pattern already established for the LLM in ADR-0002. `bge_m3.py`, `nomic.py`, `qwen_embedding.py`, `sentence_transformer_base.py`, `bge_reranker.py`, and `qwen_reranker.py` are removed (#80), along with their weights. `clip_provider.py` is kept — it is the only path to image embeddings and NIM has no equivalent multimodal model in this catalog.

## Consequences

- **Full reindex required (#79).** Switching embedding model changes the vector space; the corpus is reembedded via `rebuild_embeddings.py --recreate-collection` as a one-time offline cut (no blue-green — no uptime requirement on this project). Any future embedding provider change carries the same cost.
- **Sparse vectors move to BM25.** NIM's embedqa model is dense-only, so `search_hybrid`'s sparse side falls back to BM25 — the same pattern OpenAI/Cohere/Voyage/Gemini already use in this codebase. Hybrid search stays on; dense-only search was rejected as a retrieval-quality regression nobody asked for.
- **Asymmetric failure handling on the hot path (#78).** An embedding failure fails loud (`EmbeddingError`) — there is no fallback without a dense query vector. A reranker failure degrades gracefully — the response returns chunks in raw retrieval order instead of failing the whole request, since reranking is a quality improvement, not a requirement.
- **Redis cache recommended, not defaulted, for NIM embeddings.** `EmbeddingCacheSettings.enabled` stays `False` by default (unchanged) — flipping the default conditionally on the active provider would be hidden cross-field logic for little gain. Instead, the reference config/`.env.example` documents setting `EMBEDDINGS__CACHE__ENABLED=true` for `nvidia_nim`: every `embed_query` is a network call anyway, so caching by query text offsets repeated network latency. The reranker has no cache — pairwise scores are too query/chunk-specific to reuse. See `src/core/settings.py` for the authoritative default and the NIM dimensions caveat.
- **GPU infra can be retired later (#80, follow-up).** `helm/rag-platform/values.yaml`'s `embeddingsDevice: cuda` profile and the `models` PVC exist to support self-hosted text embeddings/reranking; once this migration ships, that GPU node pool and PVC are no longer needed for text (CLIP still needs local inference for images).
