# NVIDIA NIM provider uses system/user message roles instead of replicating llama.cpp's single-message join

`LLMRepository.generate(prompt, context, **kwargs)` takes a built system-prompt-with-context (`prompt`) and the raw user question (`context`). `LlamaCppProvider._join()` flattens both into one `role: "user"` message (`f"{prompt}\n\n{context}"`), since it's a stopgap for a chat template that doesn't reliably separate roles. The new `NvidiaNimProvider` instead sends `{"role": "system", "content": prompt}` and `{"role": "user", "content": context}` as separate OpenAI chat messages — NIM's hosted catalog models are trained to expect a real system/user split, and flattening them to match llama.cpp's local quirk would throw away the main quality benefit of switching providers. This means switching `LLM__PROVIDER` between `llama_cpp` and `nvidia_nim` changes not just infra but answer behavior — a deliberate trade-off, not an oversight.
